# Signal Dashboard (Vercel)

Static dashboard that reads `report.json` (generated by `analysis/historical_report.py`) and renders KPIs, per-condition / per-ticker win rates, and a filterable signal table.

## Deploy to Vercel (2 minutes)

### Option A — CLI

```bash
npm i -g vercel
cd dashboard
vercel                  # first-time, links the project
vercel --prod           # deploy to production URL
```

That's it — Vercel will pick up `vercel.json`, serve `index.html` at the root and `report.json` next to it.

### Option B — GitHub integration

1. Push this branch to `kushagra93/hyperliquid-perps-analyzer`.
2. https://vercel.com/new → Import the repo.
3. **Root Directory:** `dashboard`
4. **Framework Preset:** Other (none).
5. Deploy.

## Refreshing the data

The dashboard reads `dashboard/report.json` from the deployed origin. To refresh:

```bash
python3 analysis/historical_report.py --days 7
cp analysis/report.json dashboard/report.json
git add dashboard/report.json && git commit -m "data: refresh signal report"
git push                 # Vercel auto-redeploys static
```

For an automated refresh on the Oracle host, add a cron entry:

```cron
*/15 * * * * cd /home/ubuntu/hyperliquid-perps-analyzer && \
  /home/ubuntu/hyperliquid-perps-analyzer/.venv/bin/python3 analysis/historical_report.py --days 7 && \
  cp analysis/report.json dashboard/report.json && \
  git add dashboard/report.json && \
  git -c user.email=bot@oracle -c user.name=oracle-bot commit -m "data: refresh" 2>/dev/null && \
  git push origin main 2>/dev/null
```

## Live alert stream (next iteration, optional)

The MVP only shows the historical reconstruction. To stream real alerts as they fire:

1. Have the worker append each alert dict to a public JSON endpoint (Supabase / Vercel KV / a GitHub commit on a `data` branch).
2. Replace `fetch("./report.json")` with a polling fetch on that endpoint, or upgrade to Server-Sent Events.

The current `eval/alerts.jsonl` produced by `core/ticker_worker.py` is the source of truth — only thing missing is publishing it somewhere the dashboard can read.
